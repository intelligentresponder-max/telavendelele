📦 GitHub-Anleitung – Schritt für Schritt
ℹ️ Du verwendest das bestehende Repo intelligentresponder-max/telavendelele. Alle Dateien kommen in denselben Ordner – du erstellst keinen neuen Branch.
1
Dateien herunterladen (in Claude)
Lade alle 7 Dateien aus Claude herunter. Sie landen in /sdcard/Download/. Wichtig: Ordner anlegen, damit es nicht chaotisch wird:
# In Termux – einmalig vorbereiten mkdir -p /sdcard/Download/fahrzeug-agent/src/templates mkdir -p /sdcard/Download/fahrzeug-agent/.github/workflows mkdir -p /sdcard/Download/fahrzeug-agent/data
2
In dein bestehendes Repo wechseln
Das Repo telavendelele liegt bereits lokal bei dir. Öffne Termux und wechsle rein:
cd ~/telavendelele
3
Ordnerstruktur im Repo anlegen
Diese Unterordner braucht der Agent. Einmalig ausführen:
mkdir -p src/templates .github/workflows data
4
Alle Dateien ins Repo kopieren
Kopiere die heruntergeladenen Dateien an die richtigen Stellen:
# Python-Skripte cp /sdcard/Download/searcher.py src/ cp /sdcard/Download/report_builder.py src/ # Konfiguration & Abhängigkeiten cp /sdcard/Download/config.yaml . cp /sdcard/Download/requirements.txt . cp /sdcard/Download/README.md . # GitHub Action cp /sdcard/Download/daily_search.yml .github/workflows/ # Leerer Cache (Pflicht für ersten Lauf) echo '{}' > data/last_results.json
5
Alles committen und pushen
Git fügt alle neuen Dateien hinzu und schickt sie zu GitHub. GitHub fragt nach Username + Token.
git add . git commit -m "Fahrzeug-Agent hinzugefügt" git push
🔑 Kein Passwort – du brauchst einen Personal Access Token. Erstellen unter: github.com/settings/tokens → "Generate new token (classic)" → Haken bei repo
6
GitHub Action aktivieren
Die Suche läuft jetzt täglich automatisch. Einmalig manuell testen:
GitHub → Dein Repo → Reiter "Actions" → "Tägliche Fahrzeugsuche" → "Run workflow" → Klick auf grünen Button
✅ Nach dem Lauf erscheint unter "Artifacts" die Datei bericht.md mit allen Inseraten + WhatsApp-Vorlage.
7
Lokal testen (optional)
Du kannst die Suche auch direkt in Termux starten:
# Einmalig Pakete installieren pip install -r requirements.txt # Suche starten cd src python searcher.py --config ../config.yaml --output ../bericht.md
